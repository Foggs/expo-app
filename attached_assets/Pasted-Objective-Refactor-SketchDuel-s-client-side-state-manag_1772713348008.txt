Objective
Refactor SketchDuel's client-side state management from scattered useState/useRef/boolean flags into two explicit GoF-style state machines — MatchFlowMachine and TurnFlowMachine — with full error states as described in the State Pattern Migration Plan and its Addendum. The refactor must be behavior-parity first (no UX changes), incremental, and keep the existing WebSocket server protocol unchanged.

T005: Phase 1 — MatchFlow state objects
Blocked By: [T004]
Details:
Create lib/state/matchFlow/states.ts implementing IState for each of the 11 MatchFlow states. Transition rules must exactly match the migration plan and addendum:
idle: FIND_MATCH_CLICKED → connecting; others → null
connecting: WS_OPENED → queueing (effect: SEND_JOIN_QUEUE); WS_CLOSED → error_recoverable; DISCONNECT_REQUESTED → idle
queueing: QUEUE_JOINED → self (update position); MATCH_FOUND → matched; CANCEL_SEARCH → idle; WS_CLOSED(recoverable) → error_recoverable; ERROR_RAISED(fatal) → error_fatal
matched: GAME_STATE_RECEIVED → playing; CANCEL_SEARCH → idle; ERROR_RAISED(fatal) → error_fatal
playing: GAME_COMPLETE_RECEIVED → completed; OPPONENT_DISCONNECTED_RECEIVED → opponent_disconnected; WS_CLOSED(recoverable) → error_recoverable; ERROR_RAISED(fatal) → error_fatal
completed: DISCONNECT_REQUESTED → idle; all others → null
opponent_disconnected: DISCONNECT_REQUESTED → idle; all others → null
reconnecting: WS_OPENED → queueing; ERROR_RAISED(retryable, reconnectAttempt >= 8) → error_fatal; ERROR_RAISED(retryable) → error_recoverable
error_recoverable: enter stops live actions; if policy says delayed → error_backoff; else → reconnecting; DISCONNECT_REQUESTED → idle
error_backoff: enter schedules bounded exponential timer with jitter; RETRY_TIMER_EXPIRED → reconnecting; DISCONNECT_REQUESTED → idle; exit clears timer
error_fatal: enter clears tokens/session, closes socket, blocks auto-retry; only exits: DISCONNECT_REQUESTED → idle
Retry cap: 8 attempts then error_fatal. Backoff: exponential with jitter, hard max 30 seconds.
Files: lib/state/matchFlow/states.ts (new), lib/state/matchFlow/index.ts (barrel)
Acceptance: All 11 states implemented; error_fatal never schedules reconnect timers; retry cap enforced; every state exit disposes its timers.

waiting_for_turn: SERVER_TURN_CHANGED(currentPlayer === playerRole) → drawing_turn (effects: START_PLAYER_TIMER); SERVER_GAME_COMPLETED → game_complete; SERVER_OPPONENT_DISCONNECTED → opponent_disconnected; ERROR_RAISED(fatal) → sync_error_fatal

awaiting_server_ack: SERVER_GAME_STATE_ACK(currentPlayer === playerRole) → drawing_turn; SERVER_GAME_STATE_ACK(currentPlayer !== playerRole) → waiting_for_turn; ERROR_RAISED(retryable) → submit_retrying; ERROR_RAISED(fatal) → sync_error_fatal; SERVER_GAME_COMPLETED → game_complete
submit_retrying: enter pauses drawing input, keeps snapshot immutable, runs bounded retry (max 5); RETRY_BUDGET_REMAINING → awaiting_server_ack (reuse same submissionId); RETRY_BUDGET_EXHAUSTED → submit_failed; exit clears retry timer
submit_failed: enter stops retry timers, preserves unsent payload; USER_RETRY_SUBMIT → submitting_turn; SERVER_STATE_RESYNC → waiting_for_turn or drawing_turn depending on server player; ERROR_RAISED(fatal) → sync_error_fatal
sync_error_fatal: enter stops all timers/listeners, forces safe teardown; only exit: NAVIGATE_HOME
game_complete: enter triggers NAVIGATE_RESULTS effect; no further transitions
opponent_disconnected: enter triggers NAVIGATE_HOME effect; no further transitions
Retry caps: submit retries 5, then submit_failed.
Files: lib/state/turnFlow/states.ts (new), lib/state/turnFlow/index.ts (barrel)
